## Do not commit (enforced via `.gitignore`)

| Pattern | Reason |
|---------|--------|
| `node_modules/` | Install via `npm ci` |
| `.env`, `.env.*` | Secrets |
| `coverage/` | Generated by Jest |
| `assets/promo/capture-frames/` | Regenerated by `scripts/capture-app-promo.mjs` |
| `Calm Mind/`, `Calm Mind (1)/`, `Calm Mind Claude Front End Design Files/` | Local Claude Design exports; superseded by `archive/design-prototype/` and `client/js/design-*` |
| `**/.thumbnail`, `.DS_Store` | OS / tool metadata |

If duplicate design folders exist locally, delete them after confirming `archive/design-prototype/` and shipped `client/` meet your needs.

## Pre-push verification

Run from repository root:

```bash
npm ci
npm test                    # expect 112 passing
npm run launch              # smoke-test primary UI (user gesture → play)
# optional: open /index-legacy.html for legacy stack
```

Manual checks:

- [ ] No `.env`, API keys, tokens, or personal health data in the tree
- [ ] README promo GIF loads (`assets/promo/calmmind-promo.gif`)
- [ ] Primary logo loads (`assets/branding/CalmMind_Logo.jpg`)
- [ ] At least one ambient `.ogg` plays when selected
- [ ] Issue templates render on GitHub (`.github/ISSUE_TEMPLATE/`)
- [ ] CI badge in README matches default branch name (`main`)

## First upload / large migration push

1. Review `git status` — ensure design-export folders are **not** listed (gitignored).
2. Stage intentional changes only; avoid accidental `Calm Mind*` paths.
3. Use a clear commit message describing the migration (client bundle, docs layout, archive).
4. Push to `main` (or open a PR if using branch protection).

```bash
git add -A
git status   # verify no junk paths
git commit -m "Release: CalmMind client migration, docs, and GitHub packaging"
git push origin main
```

5. On GitHub: confirm **Actions** workflow `Test` runs green.
6. Set repository **About** description, topics (`web-audio`, `binaural-beats`, `wellness`), and link to README.
7. Enable **Security** → private reporting via [SECURITY.md](../../SECURITY.md) contact.

## Post-upload maintenance

When changing shipped behavior:

1. Update canonical docs per [docs/README.md](../README.md#documentation-maintenance-required).
2. Append [docs/project/changelog.md](../project/changelog.md).
3. Regenerate promo GIF if UI changed materially (`node scripts/capture-app-promo.mjs`).
4. Keep [docs/project/status-and-roadmap.md](../project/status-and-roadmap.md) accurate.

## Common pitfalls

| Pitfall | Mitigation |
|---------|------------|
| Opening `index.html` via `file://` | Always use `npm run launch` or `npm run dev` |
| Committing 40+ `capture-frames/` PNGs | Already gitignored; use committed `calmmind-promo.gif` only |
| Implying medical certification in README/issues | Keep [DISCLAIMER.md](../../DISCLAIMER.md) language |
| Forking without permission | [LICENSE](../../LICENSE) requires express written consent |
